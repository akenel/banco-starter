What this is: a walk-through that proves you can own Banco — download it, turn it on, and even rebuild a real shop from a backup — without needing the person who built it. Each step has one thing to do and a green line telling you what should happen. If it does → hit ✓ OK. If it doesn't → ✗ FAIL. If it half-worked or looked weird → ⚠ ISSUE and jot a note (type or 🎤 speak) + paste a 📋 screenshot.

You need three things only:
1 · a computer with Docker installed & running 2 · a terminal (the black typing window) 3 · ~2 GB free disk + 10 minutes
🧍 You do every step — no coding, just copy-paste commands
🛟 Can't break anything: this runs only on your own machine, in Docker. Nothing here touches any live shop. To wipe it all and start fresh: docker compose down -v. That's the reset button.
📍 A note on the web address: the steps use http://localhost:8000 (the default). If you changed a *_HOST_PORT in your .env, swap 8000 for your number (e.g. 3000). Everything else is identical.

0 · Get the code you

Download Banco to your machine. This is the whole thing — one folder.

  1. First check Docker is awake. In your terminal, type: docker --version it prints a version like "Docker version 27.x" (no red error). If it errors, start Docker Desktop first and wait for the whale icon.
  2. Now download Banco and step into the folder: git clone https://github.com/akenel/banco-starter.git cd banco-starter a banco-starter folder appears. Inside you can see compose.yml, README.md, src, scripts.

1 · Make your settings file you

Copy the example settings to a real one. The defaults already work — you don't have to change anything to start.

  1. Copy the example: cp .env.example .env a new file called .env now exists next to .env.example.

2 · Turn it on (one command) you · wait a few min

This starts four pieces at once: the database, the login system (Keycloak), the picture store (MinIO), and Banco itself. First time is slow — it downloads & builds. Grab a coffee. ☕

  1. Start everything: docker compose up --build -d it churns for a few minutes, then ends with green/white lines (no red "error"). You get your terminal back.
  2. Check the four pieces are running: docker compose ps you see banco-postgres, banco-keycloak, banco-minio, banco-app — all "running".
  3. Wait for Banco to say it's ready: docker compose logs -f app you see a line like ✨ HelixNet Core READY to serve requests. Then press Ctrl+C to stop watching (this does NOT stop the app).

3 · Install the change-log (one command) you

Adds the "who changed what, when" history (the audit log). One command, then it prints your links.

  1. Run it: ./scripts/standup.sh it prints ✅ Audit log installed and shows the POS + login links.

4 · Open the shop & log in you · in your browser

The moment of truth — see it in a browser and sign in.

Open the till ↗ localhost:8000/pos
  1. Open http://localhost:8000/pos in your browser. the Banco login screen appears (a real page, not an error).
  2. Log in with the demo cashier — username pam, password pam. you land inside the POS: products, a cart, checkout — a working till.

5 · Poke it — it really works you

A quick tyre-kick so you believe it's a real shop, not a screenshot.

  1. In the till, search for a product and add it to the cart. it adds, the price and total show.
  2. Open the change-log cockpit: http://localhost:8000/pos/audit. (If it asks you to log in as a manager, use ralph / ralph.) the 🕵️ Audit Log page opens — filters, a timeline, who/when/what.

6 · 🎁 BONUS — bring a REAL shop back from a backup the juicy part

This is the whole reason "you own it" matters. With a read-only backup key and a passphrase, you can pull a real shop's nightly backup out of Backblaze B2 and rebuild it here — on a machine that never touched the original. Skip this section if you don't have a key yet — the shop above still works.

🛟 Safe: the backup is decrypted and loaded only into your local database. Nothing is uploaded, nothing is changed in B2 (the key is read-only). Your .env is never shared.
  1. Open .env and fill in the four B2 lines you were given: B2_KEY_ID=… B2_APP_KEY=… B2_BUCKET=… BACKUP_GPG_PASSPHRASE=… all four lines have your values (read-only key + bucket name + the backup passphrase).
  2. Pause the app, then restore the newest backup: docker compose stop app ./scripts/restore-from-b2.sh it downloads a .sql.gz.gpg, decrypts it, reloads the database, and prints something like products=5200 (the real count).
  3. Start the app again and reopen the till: docker compose up -d app Then reopen http://localhost:8000/pos. the shop now has the real products from the backup — not the handful of demo items. You rebuilt a shop from nothing but a key.

7 · Turn it off / start over you

How to stop it, and how to wipe it clean.

  1. Stop it but keep your data: docker compose down the four containers stop. Next up picks up where you left off.
  2. …or wipe EVERYTHING and reclaim the disk: docker compose down -v containers stop and all data volumes are deleted — a totally clean slate.
